## Context

应用目前没有与系统登录项（Login Items）集成的能力，用户只能手动启动。现有设置体系已支持其他选项的持久化与 UI 展示，需要在此基础上新增一个可切换的“开机自启动”选项，并与系统登录项保持一致。

## Goals / Non-Goals

**Goals:**
- 在设置中提供“开机自启动”开关，勾选即启用、取消即关闭。
- 选项状态与系统登录项保持一致，重启后仍生效。
- 不影响现有窗口/时钟逻辑，仅新增启动行为与设置管理。

**Non-Goals:**
- 不实现跨平台自启动（仅针对 macOS）。
- 不增加复杂的权限管理 UI（依赖系统提示或失败提示即可）。
- 不改变现有设置存储结构以外的配置体系。

## Decisions

- 使用 macOS 官方 Login Items API（如 `SMAppService`）管理自启动，以确保与系统设置一致且符合系统安全策略。
- 在设置持久化中新增布尔字段 `launchAtLogin`（或同等命名），并在应用启动时同步状态。
- UI 侧以单一复选框/开关展示，修改后立即写入设置并调用系统 API 同步。

## Risks / Trade-offs

- [系统 API 行为差异] → 在不支持的系统版本上进行版本判断并给出降级处理或提示。
- [权限或系统限制导致设置失败] → 显示失败提示并回滚 UI 状态，确保与系统实际状态一致。
- [状态不同步] → 应用启动时主动读取系统登录项状态并刷新设置。
